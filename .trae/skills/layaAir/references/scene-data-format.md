# LayaAir3 场景与预制体数据文件格式

LayaAir3 的场景（`.ls`）与预制体（`.lh`）数据文件采用标准 JSON 格式。其中的键（key）分为两大类：

- **指令型键**：以 `_$` 开头，用于表达引擎可识别的特殊功能或元数据。
- **原生属性键**：除指令型键之外的所有键，对应对象在脚本中的实际属性名称。

## 一、节点对象（Node）

节点对象中可能包含以下指令型键：

### _$ver
- **类型**：数字或字符串
- 值固定为 `1`（或 `"1"`）。仅用于根节点，且必须提供，作为识别 LayaAir3 数据文件的标志。

### _$runtime
- **类型**：字符串（可选）
- 仅用于根节点，表示该节点绑定的脚本基类。路径格式为 `res://uuid`（TypeScript 文件的 UUID）。
- **重要说明**：`_$runtime` 绑定的是 **Runtime 类（脚本基类）**，其基类是目标节点的类型（例如 `Laya.Box` / `Laya.Sprite` / `Laya.Scene` 等），运行时该节点会按这个自定义类来实例化。它**不是**通过 `_$comp` 挂载的 `Laya.Script` 组件。

### _$var
- **类型**：布尔值（可选）
- 当使用了 `_$runtime` 后，可通过此属性标记是否在脚本中为该节点生成变量声明。

### _$preloads
- **类型**：字符串数组（可选）
- 仅用于根节点。定义预制体/场景载入时需同时加载的资源列表。路径格式为 `res://uuid`。

### _$preloadTypes
- **类型**：字符串数组（可选）
- 仅用于根节点，配合 `_$preloads` 使用，指明每个资源的类型。某些情况下必须提供，例如扩展名为 `.json` 的资源实为 Spine 数据，则需在此写入 `"Spine"`。

### _$id
- **类型**：字符串
- 节点的唯一标识，通常为 8 位由小写字母与数字组成的随机字符串。只需保证在当前预制体内唯一。从第三方工具导入的文件允许使用 `#`。若非预制体覆盖属性节点，则必须提供。

### _$type
- **类型**：字符串
- 节点的类型。若非预制体节点或预制体覆盖属性节点，则必须提供。可以是引擎内置类型（如 `"Image"`、`"Sprite3D"`），也可以是用户自定义脚本的 UUID。

### _$prefab
- **类型**：字符串
- 预制体资源的 UUID。若该节点为预制体实例，则必须提供此键，且无需再填写 `_$type`。

### _$override
- **类型**：字符串或字符串数组
- 仅出现在 `_$prefab` 节点的 `_$child` 数组中，表示覆盖预制体内某个节点的属性。若为字符串，则为目标节点的 `_$id`；若为长度为 n 的数组，则前 n-1 个元素表示多层嵌套预制体根节点的 `_$id`，最后一个元素表示最终要修改的节点的 `_$id`。

### _$parent
- **类型**：字符串或字符串数组
- 仅出现在 `_$prefab` 节点的 `_$child` 数组中，表示新增节点，将挂载到预制体的某个节点下。格式规则同 `_$override`。

### _$index
- **类型**：整数（可选）
- 与 `_$parent` 搭配使用，指定在目标父节点子级列表中的插入位置。

### _$child
- **类型**：对象数组（可选）
- 该节点的子节点列表。

### _$comp
- **类型**：对象数组（可选）
- 该节点上挂载的组件列表。

### 节点类型速查

| 类型 | 关键指令键 | 说明 |
|------|-----------|------|
| 普通节点 | `_$id`、`_$type` | 需提供唯一 ID 与类型 |
| 预制体节点 | `_$id`、`_$prefab` | 预制体实例，可在 `_$child` 中通过 `_$override` 覆盖属性或 `_$parent` 新增子节点 |
| 属性覆盖节点 | `_$override` | 覆盖预制体内部节点属性，必须位于 `_$prefab` 节点的 `_$child` 中 |
| 新增节点 | `_$parent`、`_$index` | 在预制体内部新增节点，同时需 `_$type`（通常也有 `_$id`），必须位于 `_$prefab` 节点的 `_$child` 中 |
| 根节点 | `_$ver`（必须）、`_$runtime`/`_$preloads`/`_$preloadTypes`（可选） | 必须包含 `_$ver: 1`，可选绑定脚本与预加载资源 |

## 二、组件对象（Component）

### _$type
- **类型**：字符串
- 组件的类型。若非预制体覆盖属性节点，则必须提供。可以是引擎内置类型（如 `"MeshRenderer"`、`"RigidBody"`），也可以是自定义脚本的 UUID。

### _$id
- **类型**：字符串
- 组件的唯一标识，格式同节点 `_$id`。单例组件（同一节点仅允许挂载一个）不应提供此键；非单例组件必须提供。

### _$override
- **类型**：字符串
- 仅出现在 `_$prefab/_$comp` 数组中，或出现在 `_$prefab/_$child/_$override/_$comp` 数组中。值为目标组件的 `_$type` 或 `_$id`（视目标组件是否为单例而定）。

### 组件类型速查

| 类型 | 关键指令键 | 说明 |
|------|-----------|------|
| 单例组件 | `_$type` | 只可添加一次，只需提供类型 |
| 非单例组件 | `_$id`、`_$type` | 需要提供唯一 ID 和类型 |
| 属性覆盖组件 | `_$override` | 覆盖预制体内部组件属性，仅出现在预制体相关的 `_$comp` 数组中 |

## 三、特殊用途指令键

### 资源引用
使用 `_$uuid` 与 `_$type` 搭配，表示对某个资源的引用：
```json
{
  "texture": {
    "_$uuid": "3b15da0e-f2e4-4c83-92d2-bee8c233b1f7",
    "_$type": "Material"
  }
}
```

**注意**：这种引用方式仅适用于属性类型是对象的情况（如 Texture、Material、Prefab）。如果属性类型是字符串，则属性值为 `res://xxxx` 格式（xxxx 是资源 UUID），不使用上述格式。

### 节点引用
使用 `_$ref` 引用同一文件内的某个节点：
```json
{ "refNode": { "_$ref": "wov6aw5d" } }
```

### 组件引用
使用 `_$ref` 与 `_$type` 引用某节点上的特定组件：
```json
{ "refComponent": { "_$ref": "wov6aw5d", "_$type": "MeshFilter" } }
```

### 控制器引用
使用 `_$ref` 与 `_$ctrl` 引用某节点上的控制器：
```json
{ "refController": { "_$ref": "wov6aw5d", "_$ctrl": "c1" } }
```

### 模板节点引用
常用于列表类控件（如 List、GList），通过 `_$ref` 与 `_$tmpl` 指定列表项的模板节点：
```json
{ "_templateNode": { "_$ref": "wov6aw5d", "_$tmpl": "itemTemplate" } }
```

## 四、场景 / UI 实践约定

### 场景层级约定（2D/3D 混合）
- **根节点**通常为 `Scene`，建议命名 `Scene2D`（同时具备 `_$ver/_$id/_$type` 等根节点字段）。
- **包含 3D 时**：把 `Scene3D` 放在根节点 `_$child` 的第一个子节点，所有 3D 节点统一放到 `Scene3D._$child` 下。
- **UI/2D 显示节点**：放在根节点 `_$child`（不要放进 `Scene3D`）。

### UI 组件类型（新旧 UI 区分）
- **新版 UI**：类型名前带 `G`，例如 `GBox`、`GList`、`GLabel`、`GImage` 等。
- **经典 UI**：不带 `G`，例如 `Box`、`List`、`Label`、`Image` 等。
- **场景与代码一致**：场景节点使用新版 UI（如 `GBox`），代码中必须用 `Laya.GBox`；使用经典 UI（如 `Box`），代码中用 `Laya.Box`，不可混用。

## 五、新版 UI 类型说明

### 行为组件

GLabel（标签）、GButton（按钮）、GComboBox（下拉框）、GProgressBar（进度条）、GSlider（滑动条）、GScrollBar（滚动条）

这类组件应用了组合设计模式，将显示和行为分离。组件只实现行为逻辑，不带显示功能，需要绑定显示组件才能正常工作。

例如，GButton 创建后在舞台上完全空白，只有绑定 GTextField 作为标题组件或 GImage 作为图标组件后，才会具有按钮的实际效果。通常会在 IDE 中制作成预制体再使用。

这种设计的好处是可以自由定制行为组件的显示部分，直接设置显示组件的所有属性，实现高度定制化。

### 最佳实践

1. 先确定用的是新版 UI 还是经典 UI（新版 UI type 有 `GBox/GList/GLabel/GImage` 等，规则与 FairyGUI 相似；经典 UI 为 `Box/List/Label/Image` 等）
2. 选定需要的 UI 元素后，通过 `get_schema_by_name` 查询属性 schema，严格遵守 schema 的类型约束
3. 使用 `Laya_EditAsset` 对场景/预制体数据进行修改
